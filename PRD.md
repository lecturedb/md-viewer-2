# PRD: React 실시간 마크다운 뷰어

## 1. 프로젝트 개요

React 기반의 미니멀한 실시간 마크다운 뷰어 웹페이지를 구현한다. 사용자는 좌측 입력 영역에 마크다운을 작성하고, 우측 출력 영역에서 렌더링 결과를 즉시 확인할 수 있다. 데스크톱에서는 입력과 미리보기를 나란히 배치하고, 모바일에서는 가독성을 유지할 수 있도록 상하 배치 또는 탭 전환 UI를 제공한다.

## 2. 목표

- 마크다운 작성과 결과 확인을 한 화면에서 빠르게 수행할 수 있게 한다.
- 핵심 마크다운 문법을 읽기 좋고 안정적으로 렌더링한다.
- 밝고 깔끔한 미니멀 UI로 학습 비용이 낮은 사용 경험을 제공한다.
- 데스크톱과 모바일 모두에서 입력 및 미리보기 흐름이 자연스럽게 유지되도록 한다.

## 3. 대상 사용자

- 간단한 문서 초안, README, 메모를 빠르게 확인하고 싶은 사용자
- 마크다운 문법을 배우거나 테스트하는 사용자
- 별도 계정이나 복잡한 편집기 없이 가벼운 미리보기 도구가 필요한 사용자

## 4. 핵심 기능

### 4.1 마크다운 입력

- 좌측 패널에 다중 행 텍스트 입력 영역을 제공한다.
- 사용자가 입력한 내용은 별도 저장 버튼 없이 즉시 상태에 반영된다.
- 입력 영역은 긴 문서 작성이 가능하도록 충분한 높이와 스크롤을 지원한다.
- 기본 예시 마크다운을 초기값으로 제공해 첫 화면이 비어 보이지 않게 한다.

### 4.2 실시간 미리보기

- 우측 패널에 입력된 마크다운의 렌더링 결과를 즉시 표시한다.
- 입력 변경 시 페이지 새로고침 없이 React 상태 변경만으로 미리보기를 갱신한다.
- 입력이 비어 있을 경우 빈 상태 메시지 또는 빈 미리보기 영역을 깔끔하게 표시한다.

### 4.3 지원 마크다운 문법

다음 문법은 필수로 읽기 좋게 표시되어야 한다.

- 제목: H1, H2, H3, H4, H5, H6
- 문단
- 블릿 목록
- 숫자 목록
- 인라인 코드
- 코드 블록

권장 지원 범위:

- 굵게, 기울임, 링크, 인용문, 구분선
- GitHub Flavored Markdown 일부 문법은 라이브러리 도입 여부에 따라 선택적으로 지원한다.

### 4.4 레이아웃

데스크톱 기준:

- 전체 화면은 좌측 입력 영역과 우측 결과 출력 영역으로 구성한다.
- 두 영역은 5:5 비율을 기본으로 한다.
- 각 패널은 독립적으로 스크롤될 수 있어야 한다.
- 패널 제목 또는 간단한 라벨을 제공해 영역을 명확히 구분한다.

모바일 기준:

- 화면 폭이 좁을 때는 상하 배치를 기본으로 한다.
- 대안으로 입력/미리보기 탭 전환 방식을 사용할 수 있다.
- 모바일에서도 텍스트 입력, 코드 블록, 목록이 잘리지 않고 읽혀야 한다.

### 4.5 디자인

- 전체 UI는 밝은 배경 기반의 미니멀 스타일을 적용한다.
- 색상은 흰색, 연한 회색, 짙은 회색 텍스트를 중심으로 사용한다.
- 패널 경계, 구분선, 포커스 상태는 과하지 않게 표현한다.
- 입력 영역은 문서 작성에 적합한 고정폭 또는 가독성 높은 폰트를 사용한다.
- 미리보기 영역은 실제 문서처럼 읽기 좋은 행간, 여백, 제목 계층을 가진다.
- 코드 블록은 배경색, 패딩, 둥근 모서리, 가로 스크롤을 적용해 읽기 좋게 표시한다.

## 5. 비기능 요구사항

### 5.1 성능

- 일반적인 짧은 문서와 README 수준의 입력에서는 입력 지연이 체감되지 않아야 한다.
- 렌더링은 클라이언트에서 즉시 수행한다.
- 불필요한 네트워크 요청 없이 동작해야 한다.

### 5.2 접근성

- 입력 영역에는 명확한 라벨을 제공한다.
- 키보드만으로 입력 영역과 모바일 탭 또는 주요 컨트롤을 사용할 수 있어야 한다.
- 텍스트와 배경의 대비는 읽기에 충분해야 한다.
- 포커스 상태가 시각적으로 구분되어야 한다.

### 5.3 반응형

- 데스크톱, 태블릿, 모바일 폭에서 레이아웃이 깨지지 않아야 한다.
- 작은 화면에서 코드 블록과 긴 단어는 가로 스크롤 또는 줄바꿈으로 처리한다.
- UI 요소의 텍스트가 버튼이나 패널 밖으로 넘치지 않아야 한다.

### 5.4 유지보수성

- 마크다운 렌더링은 검증된 라이브러리 사용을 우선한다.
- 입력 상태, 레이아웃, 마크다운 스타일은 명확히 분리한다.
- 컴포넌트 구조는 작고 이해하기 쉽게 유지한다.

## 6. 권장 기술 스택

- React
- Vite
- TypeScript
- Markdown 렌더링: `react-markdown`
- 코드 블록 스타일링: CSS 기반 기본 스타일 또는 필요 시 syntax highlighter
- 스타일링: CSS Modules, 일반 CSS, 또는 프로젝트 표준에 맞는 방식

## 7. 주요 화면 구성

### 7.1 데스크톱

```text
+-----------------------------------+-----------------------------------+
| Markdown                          | Preview                           |
|                                   |                                   |
| [textarea 입력 영역]              | [렌더링 결과 영역]                |
|                                   |                                   |
+-----------------------------------+-----------------------------------+
```

### 7.2 모바일

상하 배치 예시:

```text
+-----------------------------------+
| Markdown                          |
| [textarea 입력 영역]              |
+-----------------------------------+
| Preview                           |
| [렌더링 결과 영역]                |
+-----------------------------------+
```

탭 전환 예시:

```text
+-----------------------------------+
| [Edit] [Preview]                  |
+-----------------------------------+
| 선택된 영역 표시                  |
+-----------------------------------+
```

## 8. 사용자 흐름

1. 사용자가 웹페이지에 접속한다.
2. 좌측 입력 영역에 기본 예시 마크다운이 표시된다.
3. 사용자가 마크다운을 입력하거나 수정한다.
4. 우측 미리보기 영역이 즉시 업데이트된다.
5. 사용자가 데스크톱에서는 양쪽 패널을 동시에 확인하고, 모바일에서는 상하 배치 또는 탭을 통해 입력과 결과를 확인한다.

## 9. 수용 기준

- 데스크톱 화면에서 입력 영역과 미리보기 영역이 5:5 비율로 나란히 표시된다.
- 입력 영역에 마크다운을 작성하면 미리보기 영역이 즉시 갱신된다.
- H1~H6 제목이 계층별 크기와 여백을 가지고 표시된다.
- 문단, 블릿 목록, 숫자 목록이 읽기 좋은 간격으로 표시된다.
- 코드 블록은 일반 문단과 구분되는 배경, 패딩, 고정폭 폰트를 가진다.
- 모바일 화면에서 레이아웃이 깨지지 않고 입력과 미리보기를 모두 사용할 수 있다.
- 전체 UI가 밝고 깔끔한 미니멀 스타일을 유지한다.
- 키보드 포커스와 입력 라벨이 제공되어 기본 접근성을 충족한다.

## 10. 제외 범위

초기 버전에서는 다음 기능을 필수로 포함하지 않는다.

- 사용자 계정 및 로그인
- 문서 저장, 불러오기, 동기화
- 파일 업로드 및 다운로드
- 다중 문서 관리
- 협업 편집
- 커스텀 테마 관리
- 서버 사이드 렌더링 또는 백엔드 API

## 11. 향후 개선 아이디어

- 로컬 스토리지 기반 자동 저장
- 다크 모드
- 미리보기 HTML 또는 마크다운 파일 다운로드
- 코드 블록 문법 강조
- 입력/미리보기 스크롤 동기화
- 단축키 지원
- 마크다운 문법 도움말 패널

## 12. 구현 메모

- 마크다운 파싱과 렌더링은 직접 구현하기보다 `react-markdown` 같은 라이브러리를 사용한다.
- 코드 블록은 `pre`와 `code` 스타일을 별도로 지정해 긴 줄이 UI를 깨뜨리지 않게 한다.
- 모바일 레이아웃 기준점은 768px 안팎을 우선 검토한다.
- 입력 영역과 미리보기 영역은 동일한 높이 체계를 가지되, 화면 크기에 따라 자연스럽게 확장되도록 한다.
- 초기 예시 문서는 제목, 문단, 목록, 코드 블록을 모두 포함해야 한다.
